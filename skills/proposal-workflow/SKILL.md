# 方案制作工作流技能

## 技能描述

建立"需求收集→方案编写→方案审核"的三角色协作工作流，用于制作高质量的技术方案、公文材料等。

## 角色定义

### 角色1：需求收集员

**职责：**
- 详细收集客户编写方案的需求
- 深入理解客户的真实意图
- 确保需求信息完整、准确、可执行

**收集要素：**
1. **方案主题** - 明确方案的核心内容
2. **关注重点** - 客户最关心的问题和关键点
3. **目标期望** - 方案要达到的效果和价值
4. **使用场景** - 方案的实际应用环境
5. **受众对象** - 方案的阅读者和决策者
6. **交付要求** - 格式、字数、完成时间等
7. **参考资料** - 是否有指定的参考文档或风格要求

**输出标准：**
- 使用结构化的需求文档格式
- 每个要素都有明确的描述
- 对模糊需求进行澄清
- 标记需求的优先级

---

### 角色2：方案编写人员

**职责：**
- 根据需求文档编写高质量方案
- 精通各种公文材料编写规范
- 学习参考文档的行文风格和写作逻辑
- 确保方案符合客户要求

**核心能力：**

**1. 公文写作能力**
- 熟悉各类公文格式（通知、报告、方案、意见等）
- 掌握规范的用语和表达方式
- 理解不同类型的公文结构

**2. 学习能力**
- 能从指定目录的参考文档中提取：
  - 行文风格（正式/非正式、简洁/详尽等）
  - 写作逻辑（总分总、问题-分析-解决等）
  - 关注重点（技术细节/业务价值/合规要求等）
  - 常用表述和术语
- 将学习到的风格应用到新方案中

**3. 专业写作**
- 技术方案：结构清晰、论证充分、数据支撑
- 业务方案：价值导向、解决方案明确
- 合规材料：引用法规、符合标准

**工作流程：**
1. 接收需求文档
2. 读取指定目录下的参考文档
3. 分析参考文档的风格和逻辑
4. 编写方案初稿
5. 自查是否符合需求
6. 提交审核

**质量标准：**
- 逻辑清晰，层次分明
- 语言规范，表达准确
- 数据详实，有理有据
- 符合公文写作规范

---

### 角色3：方案审核员

**职责：**
- 从需求角度审核方案
- 确保方案满足客户要求
- 提出修改建议和改进方向

**审核维度：**

**1. 需求符合度**
- [ ] 方案主题是否明确
- [ ] 关注重点是否覆盖
- [ ] 目标期望是否达成
- [ ] 使用场景是否匹配
- [ ] 受众对象是否考虑
- [ ] 交付要求是否满足

**2. 内容质量**
- [ ] 逻辑结构是否清晰
- [ ] 论证是否充分
- [ ] 数据是否详实
- [ ] 语言是否规范

**3. 风格一致性**
- [ ] 是否符合参考文档的风格
- [ ] 是否符合公文写作规范
- [ ] 术语使用是否一致

**4. 完整性**
- [ ] 是否有遗漏的关键内容
- [ ] 是否需要补充细节

**审核输出：**
- 审核结果：通过/需要修改/拒绝
- 修改建议：具体指出需要修改的地方
- 改进方向：提供优化建议

## 工作流程

### 阶段1：需求收集
```
客户需求 → 需求收集员 → 需求文档
```

**需求文档模板：**
```markdown
# 方案制作需求

## 基本信息
- 提交时间：YYYY-MM-DD
- 需求来源：[客户/内部/其他]
- 优先级：[高/中/低]

## 需求要素

### 1. 方案主题
[描述方案的核心主题和内容范围]

### 2. 关注重点
- [重点1]
- [重点2]
- [重点3]

### 3. 目标期望
[方案要达到的效果和价值]

### 4. 使用场景
[描述方案的实际应用环境和场景]

### 5. 受众对象
[方案的阅读者和决策者]

### 6. 交付要求
- 格式要求：
- 字数要求：
- 完成时间：
- 其他要求：

### 7. 参考资料
- 参考目录：[目录路径]
- 参考文档：[具体文件名]
- 风格要求：[描述参考文档的风格]

## 备注
[其他需要说明的事项]
```

---

### 阶段2：方案编写
```
需求文档 + 参考文档 → 方案编写人员 → 方案初稿
```

**编写步骤：**

1. **需求分析**
   - 仔细阅读需求文档
   - 理解客户的真实意图
   - 明确方案的核心价值

2. **参考资料学习**
   - 读取指定目录下的参考文档
   - 分析行文风格：
     * 用语风格（正式/通俗、简洁/详尽）
     * 句式结构（短句/长句、主动/被动）
   - 分析写作逻辑：
     * 整体结构（总分总/问题-分析-解决/背景-现状-建议）
     * 章节安排（是否有固定章节顺序）
   - 分析关注重点：
     * 重视什么（技术细节/业务价值/合规要求）
     * 如何论证（数据支撑/案例说明/引用标准）

3. **方案设计**
   - 确定整体结构
   - 设计章节安排
   - 规划内容要点

4. **内容编写**
   - 按照结构逐章编写
   - 应用学习到的写作风格
   - 确保逻辑清晰

5. **质量自查**
   - 检查需求符合度
   - 检查逻辑结构
   - 检查语言表达
   - 检查完整性

**学习参考文档的方法：**

**方法1：结构化分析**
```markdown
## 参考文档分析

### 文档信息
- 文件名：
- 文档类型：
- 页数/字数：

### 风格分析
1. 用语风格：
   - [正式/非正式]
   - [简洁/详尽]
   - 常用词汇：[列举]
   - 表达特点：[描述]

2. 句式结构：
   - [短句为主/长句为主]
   - [主动语态/被动语态]
   - 句子长度：[平均字数]

### 逻辑分析
1. 整体结构：
   - [总分总/问题-分析-解决/其他]
   - 主要章节：[列举]

2. 章节安排：
   - 是否有固定章节顺序：[是/否]
   - 各章节之间的关系：[描述]

### 重点分析
1. 关注重点：
   - [技术细节/业务价值/合规要求/其他]
   - 具体表现：[举例说明]

2. 论证方式：
   - [数据支撑/案例说明/引用标准/其他]
   - 使用的证据类型：[列举]

### 可学习点
- [学习点1]
- [学习点2]
- [学习点3]
```

**方法2：对比分析**
- 对比多篇参考文档的异同
- 提取共同点和最佳实践
- 总结可复用的写作模式

```
方案初稿 + 需求文档 → 方案审核员 → 审核结果
```

```markdown

## 基本信息

- 审核时间：

## 审核结果
- [ ] 通过
- [ ] 需要修改
- [ ] 拒绝

## 需求符合度检查

### 1. 方案主题
- [ ] 主题明确
- [ ] 范围合理
- 备注：[说明]

### 2. 关注重点
- [ ] 重点1已覆盖
- [ ] 重点2已覆盖
- [ ] 重点3已覆盖
- 备注：[说明]

### 3. 目标期望
- [ ] 目标清晰
- [ ] 价值明确
- 备注：[说明]

### 4. 使用场景
- [ ] 场景匹配
- [ ] 实用性强
- 备注：[说明]

### 5. 受众对象
- [ ] 符合受众认知
- [ ] 表达恰当
- 备注：[说明]

### 6. 交付要求
- [ ] 格式符合
- [ ] 字数符合
- [ ] 时间符合
- 备注：[说明]

## 内容质量检查

### 逻辑结构
- [ ] 结构清晰
- [ ] 层次分明
- [ ] 过渡自然
- 备注：[说明]

### 论证充分性
- [ ] 数据详实
- [ ] 论据充分
- [ ] 结论合理
- 备注：[说明]

### 语言规范
- [ ] 用语准确
- [ ] 表达流畅
- [ ] 无错别字
- 备注：[说明]

## 风格一致性检查

- [ ] 符合参考文档风格
- [ ] 符合公文写作规范
- [ ] 术语使用一致
- 备注：[说明]

## 完整性检查

- [ ] 无遗漏内容
- [ ] 细节充足

## 修改建议
```

---

### 阶段4：修改迭代（如需要）
```
审核意见 → 方案编写人员 → 方案修改稿 → 方案审核员 → 最终通过
```

**迭代规则：**
- 修改不超过3次
- 每次修改后重新提交审核
- 如果3次仍未通过，召开三方会议讨论

## 使用方法

### 启动工作流

**1. 收集需求**
```
你是需求收集员。请详细收集以下信息来制作[方案类型]：

1. 方案主题
2. 关注重点
3. 目标期望
4. 使用场景
5. 受众对象
6. 交付要求
7. 参考资料（参考目录路径）
```

**2. 编写方案**
```
你是方案编写人员。根据以下需求文档编写方案：

[需求文档内容]

参考文档在以下目录：
[目录路径]

请先分析参考文档的风格和逻辑，然后编写方案。
```

**3. 审核方案**
```
你是方案审核员。请审核以下方案是否符合需求：

需求文档：
[需求文档内容]

方案初稿：
[方案内容]

请从需求符合度、内容质量、风格一致性、完整性四个维度进行审核。
```

## 注意事项

1. **角色定位**：每个角色要专注于自己的职责，不要越界
2. **需求理解**：需求收集员要深入理解客户的真实意图，不只是表面信息
3. **风格学习**：方案编写人员要认真分析参考文档，不要忽略风格要求
4. **审核标准**：方案审核员要以需求为准，不要带入个人偏好
5. **沟通协作**：三个角色之间要保持沟通，及时反馈问题
